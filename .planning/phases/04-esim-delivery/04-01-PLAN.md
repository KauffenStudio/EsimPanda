       - Check `process.env.NEXT_PUBLIC_STRIPE_MOCK === 'true'` for mock mode
       - In-memory Map `provisioningState` tracks `{ status, data, order_id }` by payment_intent_id (in production this would be DB reads, but for Phase 4 dev with no Supabase connection, use in-memory state)
       - `export async function provisionEsim(paymentIntentId: string): Promise<ProvisionResult>`:
         a. Check provisioningState map — if already `provisioned` or `delivered`, return existing (idempotent)
         b. Set status to `provisioning` in map
         c. If mock mode: call `mockProvision()` (has built-in 3-5s delay)
         d. If real mode: call `createProvider().purchase(wholesalePlanId, 1)` (wholesalePlanId from order lookup — for now, use a placeholder since no DB)
         e. Encrypt activation data: `encrypt(JSON.stringify({ activation_code, smdp_address, qr_base64 }))`
         f. Build DeliveryData from NormalizedPurchase: map `activationQrBase64` -> `activation_qr_base64`, `manualActivationCode` -> `manual_activation_code`, extract smdp_address from activation code format `LPA:1$smdp$code`
         g. Store in provisioningState map with status `ready`
         h. Generate order_id: `"ORD-" + paymentIntentId.slice(-8).toUpperCase()`
         i. Return `{ status: 'ready', data: deliveryData, order_id }`
       - On error: retry up to 3 times with 2s delay between retries. If all fail, set status `failed` with error message.
