# Phase 4 — UI Design Contract

> Visual and interaction contract for the eSIM delivery flow. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (custom Tailwind v4 + Motion) |
| Preset | not applicable |
| Component library | none (hand-rolled primitives, continuing Phase 1 system) |
| Icon library | Lucide React |
| Font | Plus Jakarta Sans (primary), Inter (fallback), system-ui (system fallback) |

**Continuity:** This phase inherits the full design system from Phase 1 (01-UI-SPEC.md) and Phase 3 (03-UI-SPEC.md). All tokens, primitives, and animation contracts carry forward. This spec defines only delivery-specific additions.

## Spacing Scale

Inherited from Phase 1. No changes.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, icon-to-text gap |
| md | 16px | Default element spacing, card internal padding, form field gaps |
| lg | 24px | Section padding, card outer margin |
| xl | 32px | Layout gaps between major content blocks |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level vertical spacing |

Exceptions:
- QR code container: 200px square on desktop, 160px square on mobile. Not grid-aligned -- sized for scanability at arm's length.
- "Install eSIM" CTA button: 56px height (taller than standard 48px pay button) -- this is the single most important action in the entire product. Prominence is intentional.
- Setup guide step numbers: 28px circular badges -- sized to contain 2-digit numbers legibly at 14px font.

## Typography

Inherited from Phase 1. No changes to the scale.

| Role | Size | Weight | Line Height | Delivery Usage |
|------|------|--------|-------------|----------------|
| Body | 16px | 400 (Regular) | 1.5 | Setup guide step text, SM-DP+ address text, order receipt line items |
| Label | 14px | 400 (Regular) | 1.4 | "SM-DP+ Address" field label, "Activation Code" field label, status badges, step numbers |
| Heading | 24px | 700 (Bold) | 1.2 | "Your eSIM is Ready!" delivery heading, "Setup Guide" section heading |

**Delivery-specific type rules:**
- SM-DP+ address and activation code display: 16px / 400 Regular, `font-mono` for technical strings, `select-all` for easy copying
- "Copied!" toast feedback: 14px / 400 Regular via Sonner
- Provisioning status message: 16px / 400 Regular, `text-gray-600`, centered below Bambu animation
- Setup guide step number: 14px / 700 Bold, white text on accent circle
- Setup guide step title: 16px / 700 Bold (inline with step text for scannability)
- Email subject line (template): "Your eSIM for {destination} is ready!" -- Plus Jakarta Sans in email body, Arial fallback

## Focal Point

**Delivery page primary anchor (mobile):** The "Install eSIM" button, full-width, 56px tall, accent-filled, positioned as the first interactive element after the provisioning animation completes. The eye path flows: Bambu celebration -> "Your eSIM is Ready!" heading -> Install eSIM button. The manual codes and setup guide are secondary, collapsed below.

**Delivery page primary anchor (desktop):** The QR code, 200px square, centered in a bordered card with accent top bar. The eye path flows: Bambu celebration -> "Your eSIM is Ready!" heading -> QR code (scan with phone) -> manual codes below. The setup guide is secondary, collapsed below.

**Provisioning state anchor:** Bambu "preparing" animation centered in viewport with status text below. Single focal point -- no other interactive elements visible during provisioning.

## Animation Contract (Phase 4 Additions)

### Delivery UI Animations

| Animation | Easing | Duration | Properties |
|-----------|--------|----------|------------|
| Celebration -> Delivery morph | spring (stiffness: 300, damping: 20) | auto | Bambu success -> preparing, confetti fades, content shifts |
| Provisioning status text cycle | ease-in-out | 400ms per message | opacity: 0 -> 1 -> 0, y: 4 -> 0 -> -4, cycling through messages |
| QR code reveal | spring (stiffness: 400, damping: 20) | auto | scale: 0 -> 1, opacity: 0 -> 1, slight overshoot |
| Install button entrance | spring (stiffness: 300, damping: 25) | auto | y: 20 -> 0, opacity: 0 -> 1 |
| Manual codes expand | ease-in-out | 200ms | height: 0 -> auto, opacity: 0 -> 1 |
| Setup guide expand | ease-in-out | 300ms | height: 0 -> auto, opacity: 0 -> 1 |
| Copy button feedback | ease-in-out | 150ms | icon crossfade Copy -> Check |
| Error state entrance | ease-out | 300ms | opacity: 0 -> 1, scale: 0.95 -> 1 |
| Retry attempt indicator | ease-in-out | 200ms | pulse opacity on retry count badge |

### Bambu Delivery Animations

| Animation | Type | Config | Trigger |
|-----------|------|--------|---------|
| Preparing (wrapping gesture) | spring + loop | stiffness: 300, damping: 15, continuous | Provisioning in progress |
| Preparing -> Success transition | spring | stiffness: 400, damping: 10, crossfade poses | Provisioning completes |
| Error (apologetic wobble) | spring | stiffness: 200, damping: 20 | Provisioning failed (reuse from Phase 1) |

### Provisioning Status Messages (cycling)

Display these messages sequentially during provisioning, cycling every 3 seconds:

1. "Setting up your eSIM..."
2. "Almost there..."
3. "Preparing your connection..."

### Haptic Feedback (delivery-specific)

| Action | Pattern |
|--------|---------|
| Install eSIM button tap | success pattern (two short pulses) |
| Copy code tap | light impact |
| Provisioning complete | success pattern (two short pulses) |
| Provisioning failed | warning pattern (one long pulse) |

### Provisioning State Machine

```
[payment_success] -> [provisioning] -> [ready]
                                    -> [error] -> [retrying] -> [ready]
                                                             -> [failed]
```

| State | UI Display | Duration |
|-------|------------|----------|
| provisioning | Bambu preparing animation + cycling status messages | 3-10s (real), 3-5s (mock) |
| ready | Delivery content: QR/install + credentials + setup guide | Persistent until session ends |
| error | Bambu error + retry message + WhatsApp link | Auto-retry up to 3 times |
| retrying | Bambu preparing animation + "Retrying... (N/3)" | 3-10s per retry |
| failed | Bambu error + reassurance message + email promise + WhatsApp link | Persistent |

## Device Detection Contract

| User Agent Pattern | Device Family | Primary Action | Setup Guide |
|--------------------|---------------|----------------|-------------|
| `/iphone\|ipad/i` | ios | Install button -> `iosActivationLink` | iOS steps |
| `/samsung/i` | samsung | Install button -> `androidActivationLink` | Samsung steps |
| `/pixel/i` | pixel | Install button -> `androidActivationLink` | Pixel steps |
| `/android/i` (other) | android-other | Install button -> `androidActivationLink` | Generic Android steps |
| No mobile match | desktop | QR code display | All three guides available as tabs |

**Detection method:** `navigator.userAgent` client-side. Server-side: `headers().get('user-agent')` for initial render hint, hydrate client-side for accuracy.

**Fallback:** If device family cannot be determined, show QR code (desktop default) with all setup guides available. Always include "Need help? Chat with us on WhatsApp" link.
