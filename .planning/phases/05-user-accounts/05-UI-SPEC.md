```
┌──────────────────────────┐

├──────────────────────────┤

│                          │
│    ┌──────────────────┐  │
│    │  "Log In" (24px) │  │
│    │                  │  │
│    │  Email input     │  │
│    │  Password input  │  │
│    │  [Forgot pass?]  │  │
│    │                  │  │
│    │  [Log In] button │  │
│    │                  │  │

│    │  Create one →    │  │
│    └──────────────────┘  │
│                          │
```

- Form card: max-width 400px, centered horizontally
- Card padding: 32px horizontal, 24px vertical
- Field gap: 16px between inputs
- Bambu to card gap: 48px
- "Forgot password?" link: right-aligned below password field, 14px label, ghost style
- "No account? Create one" link: centered below button, 14px label

### Signup Page (`/[locale]/signup`)

Same layout as Login page with these differences:
- Heading: "Create Account"
- Fields: Email input, Password input (no confirm field -- per CONTEXT.md "two fields only")
- CTA: "Create Account" button
- Footer link: "Already have an account? Log in"
- On success: Bambu does happy bounce, then redirect to previous page or home

## Interaction States

### Form Fields

| State | Border | Background | Text |
|-------|--------|------------|------|
| Default | #E5E5E5 | #F5F5F5 | #000000 |
| Focus | #2979FF (2px ring) | #FFFFFF | #000000 |
| Error | #E53935 | #FFFFFF | #000000 |
| Disabled | #E5E5E5 | #F5F5F5 | #9E9E9E |

- Error message: 14px label, #E53935, appears below field with 4px gap
- Transition: border-color 150ms ease-in-out

### Auth Button Loading State

- Button text replaced with Bambu loading animation (small, 24px inline)
- Button remains full width, disabled during loading
- No separate loading spinner -- Bambu loading is the project standard

### Password Field

- Show/hide toggle icon (Lucide `Eye` / `EyeOff`) at right edge of input
- Icon: 20px, gray-400, changes to gray-600 on hover
- Toggle does not submit form
- Input type switches between "password" and "text"

## Animation Contract (Phase 5 Additions)

| Animation | Type | Config | Trigger |
|-----------|------|--------|---------|
| Bambu welcoming wave | spring | stiffness: 300, damping: 15 | Login/signup page mount |
| Bambu happy bounce | spring | stiffness: 400, damping: 10 | Successful signup or conversion |
| Dropdown menu open | tween | 200ms ease-out, opacity + translateY | Avatar click |
| Dropdown menu close | tween | 150ms ease-in, opacity + translateY | Click outside / Escape |
| Conversion CTA expand | spring | stiffness: 300, damping: 15 | Password field focus |
| Conversion success collapse | spring + fade | stiffness: 300, damping: 20 | Account created |
| Auth page enter | ease-in-out | 300ms, opacity + x:20->0 | Route navigation (PageTransition) |

## Responsive Behavior

### Mobile (< 768px)

- Auth form cards: full width minus 16px horizontal padding (no max-width constraint)
- Bambu: 100px (slightly smaller than default 120px to save vertical space)
- Conversion CTA: full width within delivery page content
- User menu dropdown: full width overlay from right edge, 16px from top of header

### Desktop (>= 768px)

- Auth form cards: max-width 400px, centered
- Bambu: 120px default
- Conversion CTA: max-width matches delivery page content (inherits parent width)
- User menu dropdown: 200px, positioned below avatar, right-aligned

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Form labels | All inputs have visible `<label>` elements, associated via `htmlFor` |
| Error announcements | Error messages use `role="alert"` for screen reader announcement |
| Focus management | After auth error, focus returns to first invalid field |
| Keyboard navigation | Dropdown menu: Arrow keys navigate items, Enter selects, Escape closes |
| Color contrast | Error text (#E53935 on #FFFFFF) = 4.6:1 ratio (passes WCAG AA) |
| Touch targets | All buttons and interactive elements minimum 44px height |
| Password visibility | Show/hide toggle has `aria-label="Show password"` / `"Hide password"` |
