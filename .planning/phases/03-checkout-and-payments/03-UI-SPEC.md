# Phase 3 — UI Design Contract

> Visual and interaction contract for the checkout and payments flow. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (custom Tailwind v4 + Motion) |
| Preset | not applicable |
| Component library | none (hand-rolled primitives, continuing Phase 1 system) |
| Icon library | Lucide React |
| Font | Plus Jakarta Sans (primary), Inter (fallback), system-ui (system fallback) |

**Continuity:** This phase inherits the full design system from Phase 1 (01-UI-SPEC.md). All tokens, primitives, and animation contracts carry forward. This spec defines only checkout-specific additions.

---

## Spacing Scale

Inherited from Phase 1. No changes.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, icon-to-text gap |
| md | 16px | Default element spacing, card internal padding, form field gaps |
| lg | 24px | Section padding, card outer margin |
| xl | 32px | Layout gaps between major content blocks |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level vertical spacing |

Exceptions:
- Stripe Elements iframe inputs: Stripe controls internal padding (12px via appearance config). External spacing between Stripe Element containers uses `md` (16px) vertical gap.
- Express Checkout Element container: 16px bottom margin separating it from the "or pay with card" divider.
- Pay button: full-width at `md` (16px) horizontal page padding, 48px height (larger than standard 44px min-touch for primary action prominence).

## Typography

Inherited from Phase 1. No changes to the scale.

| Role | Size | Weight | Line Height | Checkout Usage |
|------|------|--------|-------------|----------------|
| Body | 16px | 400 (Regular) | 1.5 | Email field label, device check text, payment section labels |
| Label | 14px | 400 (Regular) | 1.4 | "Have a code?" link, VAT line item label, coupon input label, order metadata |
| Heading | 24px | 700 (Bold) | 1.2 | "Checkout" page title, "Your eSIM is ready!" success heading |
| Display | 36px | 700 (Bold) | 1.1 | Not used in checkout flow |

**Checkout-specific type rules:**
- Price display (order summary total): 24px / 700 Bold (Heading role)
- Price display (subtotal, VAT, discount lines): 16px / 400 Regular (Body role)
- Strikethrough original price: 16px / 400 Regular, `text-gray-400`, `line-through` decoration
- Savings text: 14px / 400 Regular, `text-success` (#43A047)
- Coupon error text: 14px / 400 Regular, `text-destructive` (#E53935)
- "or pay with card" divider text: 14px / 400 Regular, `text-gray-400`, uppercase, `letter-spacing: 0.05em`

## Color

Inherited from Phase 1. Phase 3 uses the established palette with these checkout-specific applications:

| Role | Value | Checkout Usage |
|------|-------|----------------|
| Dominant (60%) | #FFFFFF | Checkout page background, form backgrounds |
| Secondary (30%) | #F5F5F5 | Order summary card background, device check section background |

| Destructive | #E53935 | Payment error text, invalid coupon error, Bambu error state |
| Success | #43A047 | Coupon savings text, payment success checkmark, confetti accent color |

### Accent (#2979FF) reserved for in this phase:
- "Pay EUR X.XX" primary CTA button (fill)
- Email input focus ring
- Coupon input focus ring when active
- "Apply" coupon button (secondary variant with accent text)
- Success page redirect countdown text

### Stripe Elements color mapping:
| Stripe Variable | Design System Token | Value |
|-----------------|---------------------|-------|
| colorPrimary | success | #16a34a (green-600, Stripe focus/active states) |
| colorBackground | background | #FFFFFF |
| colorText | primary (gray-800) | #1a1a1a |
| colorDanger | destructive | #dc2626 |
| fontFamily | font-sans | "Plus Jakarta Sans", "Inter", system-ui, sans-serif |
| borderRadius | radius-card | 12px |

**Note:** Stripe Elements `colorPrimary` uses green (#16a34a) rather than the brand accent blue (#2979FF) because Stripe applies colorPrimary to input focus states and selected tabs. Green signals "payment ready" and avoids conflicting with the page's accent blue CTA. This is intentional per RESEARCH.md stripe config.

## Focal Point

**Checkout page primary anchor:** The order summary card pinned at the top of the viewport (mobile) showing plan name, price breakdown, and total. The eye path flows: order summary (what am I buying) -> email field (how to reach me) -> payment buttons (how to pay). The "Pay EUR X.XX" button at the bottom is the strongest accent-colored element, drawing the final action.

**Success page primary anchor:** Bambu celebration animation centered in viewport with confetti. Below: "Your eSIM is ready!" heading, order confirmation number, email reminder. Auto-redirect countdown at bottom.

## Animation Contract (Phase 3 Additions)

### Checkout UI Animations

| Animation | Easing | Duration | Properties |
|-----------|--------|----------|------------|
| Coupon expand/collapse | ease-in-out | 200ms | height: 0 -> auto, opacity: 0 -> 1 |
| Price strikethrough appear | ease-in-out | 200ms | opacity: 0 -> 1, text-decoration line-through |
| Savings text appear | ease-out | 300ms | opacity: 0 -> 1, y: -4 -> 0 |
| Tax "Calculating..." pulse | linear | 1500ms | opacity: 0.5 -> 1.0, infinite loop |
| Price amount update | spring (stiffness: 300, damping: 25) | auto | layout animation on number change |
| Checkout skeleton pulse | ease-in-out | 1500ms | opacity: 0.5 -> 1.0, infinite |
| Express Checkout fade-in | ease-out | 300ms | opacity: 0 -> 1 |
| Payment processing overlay | ease-out | 300ms | opacity: 0 -> 1, backdrop blur 4px |
| Pay button spinner | linear | 800ms | rotate: 0 -> 360deg, infinite |

### Bambu Checkout Animations

| Animation | Type | Config | Trigger |
|-----------|------|--------|---------|
| Processing (eating bamboo) | spring + loop | stiffness: 300, damping: 15, continuous | Payment confirmation in progress |
| Success (happy dance) | spring | stiffness: 400, damping: 10, overshoot | Payment succeeded |
| Error (apologetic wobble) | spring | stiffness: 200, damping: 20 | Payment failed |
| Confetti burst | keyframes | 40 particles, random trajectories, gravity: 0.5px/frame | Simultaneous with success Bambu |

### Haptic Feedback (checkout-specific)

| Action | Pattern |
|--------|---------|
| Pay button tap | light impact |
| Payment success | success pattern (two short pulses) |
| Payment error | warning pattern (one long pulse) |
| Coupon applied | light impact |

## Layout Contract

### Mobile (< 768px)

| Element | Position | Dimensions | Behavior |
|---------|----------|------------|----------|
| Header | Fixed top | 56px height, full width | Logo left, no nav actions (minimal checkout header) |
| BottomNav | Hidden | -- | Not shown during checkout flow (reduce distractions) |
| Checkout content | Below header | full width, max-width 480px centered, 16px horizontal padding | Single-column scroll |
| Order summary | Scroll position: top of content | full width within padding | Sticky on desktop, scrolls on mobile |
| Email field | Below order summary | full width | 16px gap above |
| Device check | Below email | full width | 16px gap above, auto-populated shows inline badge |
| Coupon section | Below device check | full width | 16px gap above, collapsed by default |
| Express checkout | Below coupon | full width | 16px gap above, hidden if no wallet methods |
| Payment divider | Below express | full width | 16px vertical margin |
| Card payment | Below divider | full width | Stripe PaymentElement fills width |
| Pay button | Below card | full width, 48px height | 24px gap above, sticky bottom on mobile with 16px padding + safe area |
| Processing overlay | Full viewport | 100vw x 100vh | Centered Bambu + backdrop blur |
| Success page | Full viewport | centered content, max-width 480px | Bambu at top, text below, redirect at bottom |

### Desktop (>= 768px)

| Element | Position | Dimensions | Behavior |
|---------|----------|------------|----------|
| Header | Fixed top | 56px height, full width | Standard header with nav |
| BottomNav | Hidden | -- | Desktop uses header nav |
| Checkout content | Below header, centered | max-width 480px, centered horizontally | Single column (no sidebar split -- checkout is deliberately narrow for focus) |
| Order summary | Top of content | full width within 480px | Position: sticky, top: 72px (header + 16px) on scroll |
| Pay button | Below card payment | full width within 480px, 48px height | Not sticky on desktop (visible in flow) |

**Design rationale:** Single narrow column on desktop (480px max) rather than a two-column split. This keeps the user focused on one task, reduces eye movement, and matches the "under 2 minutes" speed goal. The order summary sticks to the top as the user scrolls to payment.

## Stripe Elements Appearance Contract

The Stripe Elements must visually match the eSIM Panda design system. This is the exact appearance configuration:

```typescript
{
  theme: 'flat',
  variables: {
    colorPrimary: '#16a34a',
    colorBackground: '#ffffff',
    colorText: '#1a1a1a',
    colorDanger: '#dc2626',
    fontFamily: '"Plus Jakarta Sans", "Inter", system-ui, sans-serif',
    borderRadius: '12px',
    spacingUnit: '4px',
  },
  rules: {
    '.Input': {
      border: '1px solid #e5e7eb',
      padding: '12px',
      fontSize: '16px',
    },
    '.Input:focus': {
      border: '1px solid #16a34a',
      boxShadow: '0 0 0 2px rgba(22, 163, 74, 0.2)',
    },
    '.Tab': {
      border: '1px solid #e5e7eb',
    },
    '.Tab--selected': {
      borderColor: '#16a34a',
      backgroundColor: '#f0fdf4',
    },
  },
}
```

**16px font-size on all inputs is mandatory.** This prevents iOS Safari from zooming into the input on focus, which breaks the layout.
