## Interaction Contracts

### PWA Install Flow

1. User completes purchase, lands on delivery/success page
2. `beforeinstallprompt` event fires (if eligible)
3. Install banner slides up below QR code (300ms)
4. User taps "Install App" -> native browser install prompt
5. On successful install -> banner disappears, confetti optional (reuse existing CSS confetti)
6. On dismiss (X) -> banner slides down, `esim-panda-install-dismissed` set in localStorage

### Dark Mode Toggle

1. User taps Sun/Moon icon in header (existing ThemeToggle)
2. `useThemeStore.toggle()` fires
3. `document.documentElement.classList.toggle('dark')` applies
4. All `dark:` prefixed Tailwind classes activate instantly (no transition on theme swap)
5. Persisted in localStorage via Zustand persist (`esim-panda-theme`)

### Push Notification Permission

1. App detects standalone display mode (`window.matchMedia('(display-mode: standalone)')`)
2. Check localStorage for `esim-panda-notif-dismissed` and timestamp
3. If eligible -> show permission modal with 200ms fade
4. User taps "Allow Notifications" -> `Notification.requestPermission()` fires
5. If granted -> register push subscription, close modal
6. If denied -> show error copy, close modal
7. "Not now" -> set localStorage flag with timestamp, re-check after 7 days

### Notification Preferences

1. Toggles render on dashboard with persisted state (Zustand + localStorage)
2. Toggle tap -> immediate visual feedback (thumb slides 200ms)
3. State synced to push subscription topics server-side (or mock in dev)
4. No confirmation needed — toggles are instantly reversible
