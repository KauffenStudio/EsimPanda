**Inherits from:** Phase 1 UI-SPEC (01-UI-SPEC.md) — all spacing, typography, color, and animation tokens carry forward. This document specifies only Phase 6 additions and overrides.

## Spacing Scale

Inherited from Phase 1. All values are multiples of 4:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing, card internal padding |
| lg | 24px | Section padding, card outer margin |
| xl | 32px | Layout gaps between cards in grid |
| 2xl | 48px | Dashboard section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Circular gauge: 96px diameter on mobile, 120px diameter on desktop (not grid-aligned, data visualization sizing)
- Modal overlay: 24px horizontal padding on mobile, 32px on desktop
- Tab bar underline: 2px height (decorative, not spacing)

## Typography

Inherited from Phase 1. No new roles added.

| Role | Size | Weight | Line Height | Phase 6 Usage |
|------|------|--------|-------------|---------------|
| Body | 16px | 400 | 1.5 | Card descriptions, purchase history details, modal body text |
| Label | 14px | 400 | 1.4 | Status badges, "Last updated" timestamp, gauge percentage text, tab labels, expandable row metadata |
| Heading | 24px | 700 | 1.2 | "My eSIMs" page title, "Purchase History" tab heading, top-up modal title |
| Display | 36px | 700 | 1.1 | Not used in Phase 6 (dashboard is interior page) |

Additional type treatments (using existing roles):
- Gauge center number (data remaining): 24px / 700 (Heading role reused)
- Gauge unit label ("GB left"): 14px / 400 (Label role)
- eSIM card destination name: 16px / 700 (Body size, Bold weight — exception to 2-weight rule justified by card hierarchy)

| Property | Value |
|----------|-------|
| Diameter | 96px (mobile), 120px (desktop) |
| Stroke width | 8px |
| Track color | #E5E5E5 (gray-200) |

| Progress color (10-20%) | #FB8C00 (warning) |
| Progress color (< 10%) | #E53935 (destructive) |
| Stroke linecap | round |
| Rotation | -90deg (start from top) |
| Center text (primary) | Data remaining in GB, 24px / 700 |
| Center text (secondary) | "GB left", 14px / 400, gray-600 |
| Animation | Progress ring animates from 0 to value on mount, spring stiffness: 300, damping: 20, duration ~600ms |
| Expired state | Full gray track, no progress, center text: "Expired", 14px / 400, gray-400 |

## eSIM Card Layout

### Mobile (< 768px) — Full Width

```
+------------------------------------------+
| [Flag] Portugal              [Active] |
|                                          |
|       +----------+                       |
|       | Circular |    Expires: May 15     |
|       |  Gauge   |    1.2 GB of 5 GB used |
|       +----------+                       |
|                                          |
|  [ Top Up ]                              |
+------------------------------------------+
```

- Card: full width, 16px padding, 10px border-radius, surface background (#F5F5F5)
- Flag: 24px inline image (country flag emoji or SVG)
- Destination name: 16px / 700
- Status badge: top-right, pill shape, 14px label
- Gauge: centered horizontally, 96px diameter
- Expiry text: 14px / 400, gray-600, right of gauge
- Usage text: 14px / 400, gray-600, below expiry
- "Top Up" button: primary accent, full width at card bottom, 44px height

### Desktop (>= 768px) — Grid Cards

- Grid: 2 columns at 768px, 3 columns at 1024px
- Card width: fill column
- Gauge: 120px diameter
- Same internal layout, proportionally spaced
- Grid gap: 24px (lg token)

### Card Sort Order

1. Active eSIMs first, sorted by expiry date ascending (soonest expiring at top)
2. Pending eSIMs next
3. Expired eSIMs last, sorted by expiry date descending (most recently expired first)

## State Machine

### Dashboard States

| State | Trigger | Display |
|-------|---------|---------|
| loading | Page mount, user authenticated | DashboardSkeleton |
| empty | Fetch complete, 0 eSIMs | BambuEmpty + "Browse Plans" CTA |
| populated | Fetch complete, >= 1 eSIM | eSIM cards grid + tabs |
| error | Fetch failed | BambuError + "Something went wrong" + "Try Again" button |
| unauthenticated | No session detected | Redirect to /login?next=/dashboard |

### Usage Refresh States

| State | Trigger | Display |
|-------|---------|---------|
| fresh | Data fetched < 5 minutes ago | "Last updated: just now" in gray-600 |
| stale | Data fetched > 5 minutes ago | "Last updated: X min ago" in warning color |
| refreshing | Manual refresh clicked | Refresh icon spins (rotate 360deg, 600ms, linear) |
| failed | Usage API returned error | Last cached data shown + amber "Usage data may be outdated" inline warning |

### Top-Up Modal States

| State | Trigger | Display |
|-------|---------|---------|
| plan-select | Modal opened | List of top-up packages for the selected eSIM |
| payment | Plan selected | Stripe PaymentElement + "Pay [amount]" CTA |
| processing | Payment submitted | BambuLoading + "Processing your top-up..." |
| success | Payment confirmed | BambuSuccess + "Data added!" + auto-close 2s + toast |
| error | Payment failed | BambuError + error message + "Try Again" |

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
